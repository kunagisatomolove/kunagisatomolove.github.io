InformationExchangePlatform API 接口文档–用户部分

1.1 API V1 接口说明

  • 使用 HTTP Status Code 标识状态
  • 数据返回格式统一使用 JSON
  • 用户权限配置统一使用 shiro + jwt

1.1.1 支持的请求方法

  • GET（SELECT）：从服务器取出资源（一项或多项）。
  • POST（CREATE）：在服务器新建一个资源。
  • PUT（UPDATE）：在服务器更新资源（客户端提供改变后的完整资源）。
  • PATCH（UPDATE）：在服务器更新资源（客户端提供改变的属性）。
  • DELETE（DELETE）：从服务器删除资源。
  • HEAD：获取资源的元数据。
  • OPTIONS：获取信息，关于资源的哪些属性是客户端可以改变的。

1.1.2 通用返回状态说明

状态码 含义 说明
200 OK 请求成功
400 BAD_REQUEST 请求的地址不存在或者包含不支持的参数
401 UNAUTHORIZED 未授权
403 DATA_NOT_FOUND 请求的数据有误或数据不存在
404 NOT_FOUND 请求的资源不存在
406 USER_NOT_EXIST 用户不存在
407 USER_OPERATION_FAIL 用户操作失败（注册，登录，修改等）
500 INTERNAL_SERVER_ERROR 内部错误

1.1.3 主要角色与权限说明

角色 权限 说明
游客 visiter:read 游客读取
用户 user:create 用户创建
用户 user:delete 用户删除
用户 user:read 用户读取
用户 user:update 用户更新
管理员 admin:update 管理员更新
管理员 admin:read 管理员读取
管理员 admin:delete 管理员删除
管理员 admin:create 管理员创建
