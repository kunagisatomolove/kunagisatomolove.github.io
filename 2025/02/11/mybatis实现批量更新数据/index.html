mybatis实现批量更新数据

背景描述： 如果要批量更新数据库数据，一般有两种方式

  1. 在业务代码中循环遍历逐条更新
  2. 一次性更新所有数据（更准确的说是一条sql语句来更新所有数据）

逐条更新(效率低)

​ 这种方式显然是最简单，也最不容易出错的，即便出错也只是影响到当条出错的数据，而且可以对每条数据都比较可控，更新失败或成功，从什么内容更新到什么内容，都可以在逻辑代码中获取。

​ 这种方式最大的问题就是效率问题，逐条更新，每次都会连接数据库，然后更新，再释放连接资源（虽然通过连接池可以将频繁连接数据的效率大大提高，抗不住数据量大），这中损耗在数据量较大的时候便会体现出效率问题。这也是在满足业务需求的时候，通常会使用上述提到的第二种批量更新的实现（当然这种方式也有数据规模的限制）。

批量更新

​ 以下是 MyBatis 的 XML 配置片段，用于批量更新 sys_user 表中的多个用户记录。下面是对这段代码的逐行解释：

  1. <update id="updateBatchUsers" parameterType="List">:

    • 定义了一个更新操作，ID 为 updateBatchUsers，参数类型为 List，表示接受一个列表作为参数。

  2. update sys_user:

    • 指定更新的表为 sys_user。

  3. <trim prefix="set" suffixOverrides=",">:

    • 使用 <trim> 标签来处理 SQL 语句，prefix="set" 表示在生成的 SQL 语句前加上 SET，suffixOverrides="," 表示如果最后有多余的逗号，则去掉。

  4. 多个 <trim prefix="sys_... =case" suffix="end,">:

    • 这些 <trim> 标签用于生成 SQL 中的 CASE 语句，根据不同条件更新字段的值。每个字段的更新都被包裹在 CASE 语句中，只有当条件符合时才会进行更新。

  5. <foreach collection="list" item="item" index="index">:

    • 遍历传入的列表 list，item 是列表中的每一项，index 是索引。

  6. <if test="item.sysStatus!=null">:

    • 如果 item 的 sysStatus 不为 null，则生成对应的 WHEN 子句。

  7. when id=#{item.id} then #{item.sysStatus}:

    • 生成的 SQL 片段，当 id 匹配时，将 sys_status 更新为 sysStatus 的值。

  8. 余下的 <trim> 标签和 <if> 条件:

    • 其他字段（如 sys_resp_msg, sys_option_type, status）的更新逻辑与 sys_status 相同，都是通过 CASE 语句来处理。

  9. user_update_time = sysdate():

    • 此行将 user_update_time 字段更新为当前时间。

  10. where id in:

    • 生成 WHERE 子句，仅对在 list 中的用户 ID 执行更新。

  11. <foreach collection="list" item="item" index="index" separator="," open="(" close=")">:

    • 遍历 list，生成 ID 列表，用于 SQL 的 IN 子句。每个 ID 用逗号分隔，并且用小括号包裹。

这段代码实现了一个批量更新用户信息的功能，根据传入的用户列表中的多个字段来更新 sys_user 表中对应用户的记录。只有在字段值不为 null 的情况下，才会进行更新。
