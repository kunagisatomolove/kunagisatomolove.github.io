DataHub中的数据合同

发表于2025-02-05|更新于2025-02-05|大数据

什么是数据协定？

数据协定是指定义不同系统、应用程序或组件之间交换的数据的结构、格式和语义的协议或规范。它是参与数据交换的各方之间的相互理解，确保数据得到正确传输和解释。

虽然合同包括协议、规范和各种结构方面，但它们的真正价值在于它们能够被验证。可验证元素包括架构、列级数据检查和运营服务级别协议

数据协定是生产者和消费者之间的协议，它明确定义了

  • 哪些数据需要从 Producer/Source 移动到 Consumer/Destination
  • 该数据的形状、其架构和语义
  • 对可用性和数据质量的期望
  • 有关合同违规和执行的详细信息

DataHub中的数据协定

在 DataHub 中，数据协定是断言的集合，或者可以对单个数据资产进行声明和强制执行的可验证内容。

数据协定中的断言围绕架构相关方面、服务等级协议 （SLA）、数据新鲜度和数据质量展开。DataHub 的 dbt 和 Great Expectations 集成允许您生成：

  • AssertionInfo 方面（定义断言的参数）
  • AssertionRunEvents（提供断言的评估结果）

作为 DataHub 的 Data Contracts 实现的一部分，我们添加了两种新的断言：

  • SLA 断言（讨论数据集何时应登陆等）
  • Schema Assertion（此处的数据应该是什么样子，其中的字段等）

在 DataHub 中实现数据协定的端到端

DataHub 中数据协定的实现旨在：

  • 数据创建者可以将数据协定编写为 YAML 文件，并将其存储在 Git 等版本控制系统中。
  • 然后，可以将这些合约部署到 DataHub，后者充当合约及其相关断言的存储库
  • 业务用户可以使用 DataHub 访问和编辑/更新数据合同。
  • 现有的数据质量工具可以评估这些断言并报告结果。

DataHub 中的数据产品和数据合同管理

在不久的将来，为了简化数据产品和数据协定的管理，您可以使用同一个 YAML 文件来定义数据产品和数据协定规范，从而允许将它们作为统一定义进行管理。这种方法确保文档和架构断言都可以作为代码进行维护，从而满足不同利益相关者的需求。

在通过 Acryl Cloud 提供的高级 DataHub 实施中，数据合约作员响应合同 – 并开始监控、执行和报告结果。

用于可扩展数据管理的 Acrory 托管 DataHub 实施

虽然 DataHub 是数据合约的基础，但 Acryl 的托管 DataHub 版本提供了大规模管理它们所需的高级工具和功能。这包括：

  • 用于生成数据协定提案的推理引擎
  • 数据生产者和使用者的审批工作流
  • 数据合同的执行机制。
